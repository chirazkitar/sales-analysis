## Overview

This project performs a full business analysis of 4 years of retail order data (9,994 order lines, ~5,009 unique orders across the US) to answer one core question: **which product categories and regions drive profitability, and what causes underperformance?**

The analysis covers the complete data workflow : cleaning, exploration, SQL querying, visualization, and actionable business recommendations, structured the same way a data analyst would approach it on the job.

---

## Key findings

| # | Finding | Impact |
|---|---------|--------|
| 1 | Copiers and Phones generate **43% of total profit** from only 12% of orders | High-value, low-volume, worth prioritizing in sales strategy |
| 2 | Discounts above **20% consistently produce negative margins** | A clear policy recommendation: cap discounts at 20% |
| 3 | The **West region outperforms East by ~28%** in average profit margin | Regional strategy and resource allocation need revisiting |
| 4 | The **Tables sub-category loses money in every region** | Candidate for discontinuation or supplier renegotiation |
| 5 | **Q4 (Oct–Dec)** shows a reliable sales spike each year | Inventory and staffing should be scaled up ahead of Q4 |

## How to run

### Prerequisites

- Python 3.9+
- pip
- A free [Kaggle account](https://kaggle.com) to download the dataset
- [Power BI Desktop](https://powerbi.microsoft.com/desktop) (free, Windows) to open the .pbix file

---

### Step 1 — Get the dataset

1. Go to the [Superstore dataset on Kaggle](https://www.kaggle.com/datasets/vivek468/superstore-dataset-final)
2. Download `Sample - Superstore.csv`
3. Rename it to `superstore_raw.csv`
4. Place it in the `/data` folder

> **No Kaggle account?** The dataset is also available on [data.world](https://data.world/vikas8989/superstoresales).

---

### Step 2 — Install dependencies

```bash
pip install -r requirements.txt
```

---

### Step 3 — Run the notebooks in order

```bash
jupyter notebook
```

Open and run each notebook **in sequence** — each one depends on the output of the previous:

| Order | Notebook | What it does | Output |
|-------|----------|--------------|--------|
| 1st | `01_data_cleaning.ipynb` | Loads raw CSV, fixes data types, removes nulls, adds derived columns (profit margin, ship days, etc.) | `data/superstore_clean.csv` |
| 2nd | `02_eda_and_charts.ipynb` | Runs exploratory analysis, generates 5 publication-quality charts | `visuals/*.png` |
| 3rd | `03_sql_insights.ipynb` | Loads clean data into SQLite, runs all 7 queries, prints findings | Console output |

> **Prefer not to install anything?** Upload the notebooks to [Google Colab](https://colab.research.google.com) and run them there — no local setup required.

---

### Step 4 — Run SQL queries standalone (optional)

If you want to run the SQL queries outside of Python:

1. Install [DB Browser for SQLite](https://sqlitebrowser.org/) — free, cross-platform
2. Open `data/superstore_clean.csv` → File → Import → Table from CSV → name it `orders`
3. Open any `.sql` file from the `/sql` folder and click Execute

---

### Step 5 — Explore the Power BI dashboard

The dashboard is split across 3 pages with full cross-filtering via Region, Category, and Year slicers.

To open the source file locally:
1. Download [Power BI Desktop](https://powerbi.microsoft.com/desktop) — free
2. Clone this repo and open `superstore-analysis.pbix`
3. When prompted, update the data source path to point to your local `data/superstore_clean.csv`

#### Dashboard pages

| Page | Contents |
|------|----------|
| **Overview** | KPI cards (Total Sales, Profit, Margin %, Orders) · Sales by Category · Profit by Sub-Category · Monthly Trend |
| **Analysis** | Discount vs Profit Margin scatter · Sales & Profit map by State · Monthly vs YTD Sales · YoY Growth % |
| **Segments** | Performance by Customer Segment · Order Share by Segment · Segment summary table |

#### Data model — star schema

The flat CSV was normalised into a star schema with 5 tables:

```
dates      (1) ──→ (*) orders ←── (1) products
                       ↑
customers  (1) ──→ (*) orders ←── (1) locations
```

| Table | Key column | Contains |
|-------|-----------|----------|
| `orders` | Order ID | Sales, Profit, Discount, Quantity (fact table) |
| `products` | Product ID | Product Name, Category, Sub-Category |
| `customers` | Customer ID | Customer Name, Segment |
| `locations` | Postal Code | City, State, Region, Country |
| `dates` | Order Date | Unique dates for time intelligence |
